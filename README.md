# Homes in Tule Springs | Dr. Jan Duffy Real Estate

A professional real estate website for Dr. Jan Duffy, REALTOR® with Berkshire Hathaway HomeServices Nevada Properties, specializing in the Tule Springs area of North Las Vegas, Nevada.

## Technology Stack

- **Next.js 15** (App Router, static export)
- **React 19**, TypeScript
- **Bootstrap 5**, custom CSS (Berkshire Hathaway branding)
- **RealScout** (MLS listings, home value widget)
- **Vercel** (hosting, Analytics, Speed Insights)
- **GitHub** (CI, Dependabot)
- **Cloudflare** (optional Web Analytics; DNS-only recommended when using Vercel)

## Development

```bash
npm install
npm run dev    # http://localhost:3000
npm run build  # static export to out/
npm run lint   # ESLint
```

## Free-Tier Setup: Vercel + GitHub + Cloudflare

### 1. Vercel (hosting + analytics)

- **Deploy:** Connect this repo in [Vercel](https://vercel.com) → New Project → Import. Push to `main` triggers production deploy.
- **Speed Insights:** Free on Hobby plan. Enable in Project → Speed Insights. Already wired in the app via `@vercel/speed-insights`.
- **Analytics:** Free tier available. Enable in Project → Analytics. Already wired via `@vercel/analytics`.
- **Headers:** Security and cache headers are in `vercel.json` and apply at the edge.

### 2. GitHub (CI + dependency updates)

- **CI:** `.github/workflows/ci.yml` runs `npm ci`, `npm run lint`, and `npm run build` on every push and pull request to `main`. Fix failures before merging.
- **Deploy (optional):** `.github/workflows/deploy-vercel.yml` deploys to Vercel on push to `main` using GitHub Actions. Requires repo secrets: `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`. If Vercel Git integration is connected, you can rely on that instead.
- **Dependabot:** `.github/dependabot.yml` opens weekly PRs for `npm` and `github-actions` updates. Review and merge to keep dependencies current.

### 3. Cloudflare (DNS + optional analytics)

- **DNS:** Use Cloudflare for the domain with **DNS only (gray cloud)** when the site is hosted on Vercel to avoid SSL/proxy conflicts.
- **Web Analytics (optional):** Free, privacy-first, works without proxying traffic.  
  1. [Cloudflare Dashboard](https://dash.cloudflare.com) → Web Analytics → Add a site → copy the beacon token.  
  2. In Vercel: Project → Settings → Environment Variables → add `NEXT_PUBLIC_CF_WEB_ANALYTICS_TOKEN` with that token.  
  3. Redeploy. The app already includes the Cloudflare beacon when this variable is set.

## SEO & Metadata

- **Sitemap:** `/sitemap.xml` is generated by `app/sitemap.ts`.
- **Robots:** `/robots.txt` is generated by `app/robots.ts`.
- **Security:** `/.well-known/security.txt` is in `public/.well-known/security.txt` (Contact, Expires).
- **404:** Custom `app/not-found.tsx` with links to Home, Listings, and phone.
